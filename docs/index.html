TCH099 Demo Web App

This repo will be used as demo PHP web app to deploy on Azure cloud

Features

  • API Endpoints: Create custom API endpoints for your application.
  • User Authentication: Implement user authentication with login, signup, and logout functionality.
  • Database Integration: Use MySQL database for storing user data and other information.
  • Frontend Pages: Create frontend pages with HTML, CSS, and JavaScript.
  • Docker Support: Run the application locally using Docker Compose.

Epic: User Account and Profile Management

User Story: User Signup

As a user, I want to create an account on the platform so that I can access the features.

Sub-tasks:

  • Design the signup form with fields for username, email, password, role.
  • Implement backend logic to handle user data submission and validation.
  • Integrate password encryption for secure storage.
  • Test the signup process to ensure it works as expected.

User Story: User Login

As a user, I want to log in to my account so that I can access my profile and other features.

Sub-tasks:

  • Design the login form with fields for username/email and password.
  • Implement backend authentication logic to verify user credentials.
  • Create sessions to maintain user login state.
  • Handle incorrect login attempts with appropriate error messages.
  • Test the login process for functionality and security.

User Story: User Logout

As a user, I want to log out of my account to secure my session.

Sub-tasks:

  • Add a logout option in the user interface.
  • Implement backend logic to destroy user sessions upon logout.
  • Redirect users to the homepage or login page after logout.
  • Test the logout functionality to ensure sessions are properly terminated.

User Story: View Hello World Message

As a user, I want to see a "Hello World" message with my name.

Sub-tasks:

  • Create a personalized message template.
  • Retrieve the user's name from the session data.
  • Display the "Hello World" message with the user's name on the page.
  • Test the message display for different user scenarios.

User Story: View Random Image

As a user, I want to see a random image displayed on the page.

Sub-tasks:

  • Source a collection from external API of random images.
  • Implement logic to randomly select an image from the collection.
  • Display the selected image on the user interface.
  • Test the random image feature to ensure different images are shown each time.

User Story: View User Details

As a user, I want to view my profile details such as username and email address.

Sub-tasks:

  • Create a user profile page template.
  • Implement backend logic to retrieve user details from the database.
  • Display the user's username and email address on the profile page.
  • Test the profile page to ensure accurate data display.

User Story: Admin Login

As an admin, I want to log in to my account to access admin features like viewing all clients.

Sub-tasks:

  • Design the admin login form with fields for username/email and password.
  • Implement backend authentication logic for admin credentials.
  • Create admin sessions to maintain login state.
  • Develop the admin dashboard to display client information.
  • Test the admin login and dashboard functionalities for proper access control.

Environment Variables

Create a .env file in your project root with the following content:

DB_HOST=tch099-db
MYSQL_ROOT_PASSWORD=rootpassword
MYSQL_DATABASE=mydatabase
MYSQL_USER=user
MYSQL_PASSWORD=password

Follow the prompts provided by Vercel CLI to complete the deployment.

Docker Usage

To run the application locally using Docker:

docker-compose up -d --build

API Endpoints

User Authentication

Login

Endpoint: /api/login.php

Method: POST

Request Body:

{
  "username": "example",
  "password": "password"
}

Response:

{
  "status": "success",
  "message": "Login successful"
}

Signup

Endpoint: /api/signup.php

Method: POST

Request Body:

{
  "username": "example",
  "password": "password",
  "email": "example@example.com",
  "role": "user"
}

Response:

{
  "status": "success",
  "message": "Signup successful"
}

Logout

Endpoint: /api/logout.php

Method: POST

Response:

{
  "status": "success",
  "message": "Logout successful"
}

User Details

Endpoint: /api/user_details.php

Method: GET

Response:

{
  "status": "success",
  "data": {
    "username": "example",
    "email": "example@example.com",
    "role": "user"
  }
}

Get Greeting

Endpoint: /api/hello-world/$langue

Method: GET or POST

Request Query Parameters:

  • nom (optional)
  • prenom (optional)

Response:

{
  "nom": "Doe",
  "prenom": "John",
  "langue": "en",
  "message": "Hello World John Doe!"
}

Random Image

Endpoint: /api/random-image

Method: GET

Response: A random image URL.
